Python Packages

Python packaging is hard. One thing that can make it hard is understanding the version numbers. Sure, Django v1.6.0 is ahead of Django v1.5.0, but what happens if you're using something like Django==1.5rc1 and you install a package that relies on Django>=1.4.1?

A peck of PEPs

There are number of PEPs that govern the way python versions ought to behave. If you stick to them, you will be doing yourself (and the community) a favor. If you find instances of tools that don't follow them, please submit an issue against this repo so that I can improve these docs. Also, please tell developers of those tools.
